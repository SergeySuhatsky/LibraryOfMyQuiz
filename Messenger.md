# Что реализовано

- **Аутентификация (сессии):** вход/регистрация через сессии — session.go.  
- **Middleware авторизации:** проверка доступа к эндпоинтам — widdleware.go.  
- **Регистрация/логин/выход:** формы и обработчики — auth.go, страницы auth.html, registration.html.  
- **Профиль пользователя:** просмотр и редактирование профиля, загрузка аватара — handlersProfile.go, api.go, фронтенд — profile.html.  
- **Стена пользователя:** CRUD постов (создать, список, редактировать, удалить) — handlersProfile.go и соответствующий UI.  
- **Друзья:** добавление/список друзей, поиск пользователей — handlersProfile.go, handlersFriend.go.  
- **Личное общение (1:1):** реальное время через WebSocket, хранение сообщений и загрузка истории чата — websocket.go, api.go.  
- **Фронтенд:** статические страницы + Vue компонентные скрипты (`/js/*`) — готов базовый SPA-поведение для профиля/чата.  
- **Статические файлы/сервер:** отдача статичных ресурсов и маршрутизация — main.go.  
- **Модели/структуры:** структуры пользователя, сообщения, стен — models.go.

# Чего не хватает
- **Групповые беседы:** невозможность создавать беседы более чем между двумя участниками.  
- **Передача медиа в чатах:** сейчас только текст; обмен фото/видео в сообщениях не реализован (есть только аватар).  
- **Управление вложениями и хранилище:** нет продуманного хранилища/CDN для медиа, версий/ограничений по размеру, транскодинга.  
- **Уведомления и статусы:** нет push/браузерных уведомлений, статус онлайн/оффлайн, индикаторы «прочитано», «печатает».  
- **Модерация и безопасность контента:** блокировка/репорты пользователей, фильтрация контента и ограничения по злоупотреблениям отсутствуют.  
- **Приватность и настройки:** управление видимостью стены, блокировка, настройки приватности отсутствуют.  
- **Поиск и рекомендация:** расширенный поиск людей/чатов, сортировка, рекомендации друзей.  
- **Масштабируемость и надежность:** отсутствие очередей/горизонтального масштабирования для WebSocket, резервного хранения и бэкапов.  
- **Управление сессиями/авторизацией:** восстановление пароля, подтверждение email, MFA — нет.  
- **Аудит/логирование/мониторинг:** базовое логирование есть, но нет метрик, алертов, системы логов для продакшна.  
- **Документация и API контракт:** нет явной документации API/контрактов для фронта/теста.  
- **Юридические/соответствие:** политики хранения данных, экспорт/удаление данных пользователя, GDPR/правила — не проработаны.  
- **UX/мобильность:** адаптивность, мобильные клиенты и удобство мобильного взаимодействия не описаны.

# Рекомендации и приоритеты (бизнес‑приоритет)
- **Приоритет 1 — «Ядро продукта»:**
  - Добавить обмен медиа в чатах (фото/видео) и хранение вложений — ключ для удобного общения.
  - Добавить групповые чаты/беседы (создание, управление участниками, администраторы).  
  - Реализовать базовые уведомления (браузерные) и индикатор непрочитанных сообщений.
- **Приоритет 2 — «Надёжность и безопасность»:**
  - Внедрить восстановление пароля и подтверждение email; в перспективе MFA.  
  - Ограничения на размер/тип файлов, проверка и сканирование вложений.  
  - Перенести медиа на отдельное надёжное хранилище (S3/аналог) и подготовить CDN‑шаблон.
- **Приоритет 3 — «UX и рост»:**
  - Онлайн/оффлайн статус, «печатает», метки «прочитано» — повышают вовлечённость.  
  - Поиск пользователей и удобная навигация по беседам.  
  - Мобильная адаптация UI и минимально приемлемый мобильный поток авторизации/чата.
- **Приоритет 4 — «Операции и соответствие»:**
  - Документация API, контракт фронт/бэк; базовые тесты (юнит/интеграция).  
  - Логирование, метрики (uptime, ошибки, нагрузка), план бэкапов БД.  
  - Политики хранения данных и возможность удаления аккаунта/данных.
- **Большая фича/долгосрочно:** end‑to‑end шифрование для приватных чатов (если бизнес требует максимальной приватности) — высокая сложность, большая ценность для доверия.



# Приоритетные предложения по переписыванию

- **WebSocket/real‑time:** заменить текущую плоскую карту `clients` в websocket.go на централизованный hub/connection manager (очереди, mutex/конкурентность, маршрутизация сообщений, поддержка группы). Почему: повышает надёжность при росте пользователей, упрощает реализацию групповых чатов и повторных попыток доставки. Бизнес‑выгода: масштабируемая архитектура реального времени, меньше простоев при пиковых нагрузках.

- **Сессии и аутентификация:** не хранить секреты в коде, централизовать логику в модуле auth (вынести конфиг в env), добавить механизмы восстановления пароля и подтверждения email. Почему: безопасность и доверие пользователей. Бизнес‑выгода: снижает риск утечек и повышает конверсию регистраций.

- **API и обработка ошибок:** унифицировать API (JSON‑контракты), ответы и коды статуса; заменить разбросанные SQL‑вызовы в `handlers*` на слой репозитория/сервиса. Почему: облегчает поддержку фронта, тестирование и интеграции. Бизнес‑выгода: быстрее разворачивать новые фичи и меньше багов при изменениях.

- **Доступ к БД / безопасность запросов:** убрать неструктурированные `Query`/`Exec` вызовы по всему коду, использовать подготовленные выражения/транзакции и отдельный DAO-уровень. Почему: предотвращение ошибок, легче оптимизировать индексы и производительность. Бизнес‑выгода: стабильность и предсказуемость при росте объёмов данных.

- **Обработка медиа и вложений:** вместо сохранения файлов в avatars — вынести хранение в сервис хранилища (S3/аналог), добавить валидаторы размеров/типов и CDN‑подготовку. Почему: масштаб и безопасность хранения. Бизнес‑выгода: быстрый доступ к медиа, экономия диска, лучшая производительность клиентов.

- **Frontend/API разделение:** превратить текущие странички + inline Vue в чистый SPA/модульный фронт с явным API‑контрактом (фронт ↔ бэк через JSON). Почему: улучшит мобильную адаптацию, тестирование и ускорит UX‑итерации. Бизнес‑выгода: более отзывчивый продукт, легче добавлять новые клиентские платформы.

- **Логирование и мониторинг:** заменить простые лог‑файлы на структурированные логи/метрики и health‑endpoints. Почему: раннее обнаружение проблем. Бизнес‑выгода: меньше простоев, данные для приоритизации багфиксов.

- **Безопасность и модерация:** ввести валидацию входных данных, rate limiting и базовые механизмы жалоб/блокировок пользователей. Почему: уменьшение злоупотреблений и правовых рисков. Бизнес‑выгода: защищает репутацию сервиса и пользователей.

- **Тесты и CI:** добавить unit/integration тесты для критичных сервисов (auth, WebSocket, загрузки файлов), настроить CI. Почему: ускоряет выпуск и уменьшает регрессии. Бизнес‑выгода: ниже стоимость поддержки, быстрее фичеролы.

# Коротко о «что начать первым»
- Быстрая победа: API‑унификация + слой репозитория (переписать 3–5 наиболее используемых эндпоинтов). Это даст быстрый ROI: фронт станет стабильнее и подготовит почву для медиа/чатов.  
- Важная инвестиция: hub для WebSocket + медиа‑хранилище — критично для конечного продукта (много общения и обмена файлами).
